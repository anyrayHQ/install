# Anyray Railway Template

Deploy Anyray to Railway with one click. Railway auto-generates all secrets
and provides managed Postgres and Redis.

## Deploy

1. Click "Deploy on Railway" on the Anyray docs page, or open the template URL
   directly (see the main README for the link once the template is published).
2. Railway will prompt you to create a new project. Accept.
3. All secrets are auto-generated by Railway's `${{secret()}}` generators.
   No manual secret entry needed.
4. Wait for all services to deploy (~3 min).
5. Open the `proxy` service → "Settings" → "Public Networking" → Generate domain.
   This is your console URL.
6. Open the `gateway` service → "Settings" → "Public Networking" → Generate domain.
   This is your gateway API URL (use this in `npx anyray-connect`).

## Seeding model prices (one-time, after first deploy)

Railway has no native one-shot Job. Run the seeder from the Railway CLI:

```bash
# Install Railway CLI if needed: npm install -g @railway/cli
railway login
railway run --service web \
  node -e "$(curl -fsSL https://raw.githubusercontent.com/anyrayHQ/install/main/scripts/seedModelPrices.mjs)"
```

Or open the `web` service shell in the Railway UI and run the seeder inline.

## Variables reference

| Variable                           | Source                          | Notes                                                     |
|------------------------------------|---------------------------------|-----------------------------------------------------------|
| `ANYRAY_ADMIN_TOKEN`               | `${{secret()}}` on gateway      | Printed in Railway after deploy. Gates console.           |
| `ANYRAY_CONTENT_KEY`               | `${{secret()}}` on gateway      | AES-256-GCM at-rest encryption key.                       |
| `LANGFUSE_INIT_PROJECT_PUBLIC_KEY` | `${{secret()}}` on web          | Shared with gateway as `ANYRAY_OBSERVABILITY_PUBLIC_KEY`. |
| All datastore passwords            | `${{secret()}}` on each service | Never leave Railway's environment.                        |

## Manual verification (required after template is first published)

The Railway template format is verified by deploying it. After pushing
`railway/railway.template.json` to the `anyrayHQ/install` repo:

1. Log in to Railway: <https://railway.com>
2. "New Project" → "Deploy a template" → search "Anyray" (or use the direct
   template URL from the Railway dashboard after submission).
3. Verify all 7 services start without errors.
4. Probe: `curl https://<gateway-domain>/` → expect `AI Gateway` response.
5. Probe: `curl -fso /dev/null -w "%{http_code}" https://<proxy-domain>/anyray-login`
   → expect `200`.
6. If any variable reference fails (e.g. `${{web.LANGFUSE_INIT_PROJECT_PUBLIC_KEY}}`
   not resolving), adjust the reference in `railway.template.json` and re-push.
   Railway's cross-service variable references are verified only during live deploy.

## Publishing the template to Railway's marketplace

After successful verification:

1. Go to <https://railway.com/deploy> → "Publish a template".
2. Point it at `github.com/anyrayHQ/install` and select `railway/railway.template.json`.
3. Fill in display name "Anyray", category "AI / ML", submit for review.

The "Deploy on Railway" button URL will be:
`https://railway.com/new/template?referralCode=anyray&template=<template-id>`
